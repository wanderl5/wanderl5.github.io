GDB调试常用命令汇总

# 启动/停止

# 启动

命令 含义
run/r 启动直到第一个断点
run > gdb.out 将程序输出重定向到输出文件,可避免cout内容污染输出
run 参数 指定命令行参数，效果同set args xxx

# 停止

命令 含义
run/r 启动直到第一个断点
kill 停止程序执行
quit/q 退出gdb调试

# 源代码管理

# 查看源码

命令 含义
list/l 查看当前所在行，前后1/2 listsize
l 行号/-行号 往后/前显示指定行的代码， 前后1/2 listsize
l 行号1,行号2 行号1到行号2的代码
l 文件名：行号 指定文件指定行的代码

# 源码参数

命令 含义
set listsize 数字 设置listsize
show listsize 查看一次显示的代码行数

# 源代码路径设置

命令 含义
directory/dir 目录名 添加目录到源码搜索路径中
show directory 查看源码搜索路径
directory/dir 清空添加的源码搜索路径(需细到cpp直接路径，多个文件需改路径，这方式不合适)
set substitute-path 设置替换路径set substitute-path /pub/smdev3.1 /home/cy1/smdev

# 断点

# 常规断点

命令 含义
break/b [文件名:]行号 指定文件指定行号，没有指定文件则表示当前文件
b [类名::/文件名:]函数名[(参数列表)] 在所有本函数断点,可以限定指定类名/文件名称，也可限定参数列表
b 行号 if 条件 条件断点 ，支持多条件 b 18 if (i==2 && j ==5)
b *指针地址 指定地址设置断点，比如函数
b +/-偏移量 当前断点位置，偏移多少地址设置断点
tbreak/tb 临时断点，命中一次后删除

针对参数列表的限定， 注意std::string参数，其表示比较特别

# 对应GetStr(double, string)
b GetStr(double, std::__cxx11::basic_string<char, std::char_traits<char>, std::allocator<char> >)

# 断点管理

命令 含义
info break / ib 查看断点
delete / del 清楚所有断点
del start[,end ] 指定序号或指定序号范围删除
enable/disable 断点编号 启用，不启用断点

# 观察点watch，catch

命令 含义
watch 观察表达式的值是否有变化，紧挨两句一样值的场景下不会命中
rwatch 读取指定字段时命中
awatch read/write命中 （观察何时修改比较合适）
catch throw 抛出异常时能被捕获
  • info break 可查看已打的watch断点， 或使用i watchpoints也可以
  • 可以使用enable/disable/delete进行观察点操作
  • 一般是针对全局的值进行观察;

# 步进

命令 含义
continue / c 继续
next / n [步数] 逐过程执行指定次数
step / s [步数] 逐语句执行指定次数
finish / fin 跳出当前执行函数
where 查看当前代码行数和函数

# 查看信息(断点/线程/寄存器)

命令 含义
i b 查看断点(breakpoint, watchpoint, catch)
i registers 查看寄存器
i threads 查看线程
i args 查看参数
i locals 查看当前作用域的局部变量
i display 查看当前自动展示变量

# 查看变量类型信息

命令 含义
ptype 变量/数据类型 查看变量或类型的详细信息
whatis 变量 查看变量的类型,原始类型
  • ptype的几点说明
    • /r: 原始数据方式展示,不代替typedef
    • /m: 不查看函数
    • /o: 打印偏移量和大小
# typedef std::string mystring;
# mystring g_str;
(gdb) whatis g_str
type = mystring
(gdb) ptype g_str
type = std::string

# 默认
(gdb) ptype /M a
type = class Clza {
  private:
    int a;
    double b;
    std::string c;
    char d;

  public:
    int getA(void) const;
    double getB(void) const;
}
# 隐藏函数
(gdb) ptype /m a
type = class Clza {
  private:
    int a;
    double b;
    std::string c;
    char d;
}
# 显示偏移量和大小, 若有虚函数,则虚函数会占用8个字节用于保存指针,多个虚函数不会成倍增加占用空间

# 查看和改变变量值

命令 含义
p [文件名:]变量名 查看指定变量
x [/nfu] 地址 指定格式查看变量
display 变量 显示变量值
  • display 至此后info查看,enable/disable/delete/undisplay来启用,删除
  • x或p的集中展现方式: | 命令 | 含义 | | -------- | -------------------- | | 不带选项 | 默认展示16进制地址 | | /[数量]c | 显示字符 | | /[数量]s | 显示字符串 | | /[数量]d/u | 查看指定数量的内存值, 10进制展示 | | /[数量]x | 从当前地址展示指定数量的16进制数 |
  • x查看的是内存布局,针对string实际上是一个类, 看到的并不仅仅是字符串,还有字符串长度等;
  • i r rdi / rsi 一般能看到函数的两个参数；

# 多线程调试

# 死锁问题跟踪

  • 死锁代码
#include <iostream>
#include <unistd.h>
# include <mutex>
# include <thread>

std::mutex _mutex1;
std::mutex _mutex2;
int data1;
int data2;

using namespace std;
void do_work_1()
{
	cout << "thread 1 begin work ..." << endl;
	lock_guard<std::mutex> locker1(_mutex1);
	data1 ++;
	std::this_thread::sleep_for(std::chrono::seconds(1));
	lock_guard<std::mutex> locker2(_mutex2);
	data2 ++;
	cout << "thread 1 work done" << endl;
}

void do_work_2()
{
	cout << "thread 2 begin work ..." << endl;
	lock_guard<std::mutex> locker2(_mutex2);
	data2 ++;
	std::this_thread::sleep_for(std::chrono::seconds(1));
	lock_guard<std::mutex> locker1(_mutex1);
	data1 ++;
	cout << "thread 2 work done" << endl;
}

int main(int argc, char **argv) {
	thread t1(do_work_1);
	thread t2(do_work_2);

	t1.join();
	t2.join();
	cout << "==== all done ==== " << endl;
    return 0;
}
