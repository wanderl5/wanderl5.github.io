C++11多线程

# C++11新标准多线程支持库

C++标准并没有提供对多进程并发的原生支持，所以C++的多进程并发要靠其他API（比如用CreateThread进行创建） C++11可以通过多线程实现并发，这是一种比较底层、传统的实现方式。

C++11引入了5个头文件来支持多线程编程，是///<condition_variable>/

库 说明
thread 提供线程创建及管理的函数或类接口；
mutex 互斥量Mutex
condition_variable 条件变量配合Mutex
future 获取异步任务返回值，并捕捉其所抛出的异常
atomic 原子变量

# std::thread类成员函数：

函数 用途
get_id 获取线程ID,返回一个类型为td::thread::id
join 通过join()函数关联并阻塞线程，等待该线程执行完毕后继续
detach 解除与主线程关联,与主线程并发，主线程退出后,detach()线程也将自动退出
joinable 检查线程是否可被join。检查thread对象是否标识一个活动(active)

# std::this_thread命名空间定义了一系列对线程的控制:

函数 用途
yield 将调用者线程跳出运行状态，重新交给操作系统进行调度
sleep_for 线程休眠某个指定的时间片(time span,结合chrono的duration使用)
sleep_until 线程休眠至某个指定的时刻(结合chrono的time point),该线程才被重新唤醒

注:

  • "joinable"的对象析构前，必须调用join()接口等待线程结束，或者调用detach()接口解除与线程的关联，否则会抛异常；
  • std::thread没有拷贝构造函数和拷贝赋值操作符，因此不支持复制操作（但是可以move），也就是说，没有两个 std::thread对象会表示同一执行线程；

# 线程使用场景-参数传递

  • 建议使用指针或引用传值,避免多次拷贝和析构
  • 引用传值需要使用std::ref包裹传递;
#include <iostream>
#include <thread>
#include <list>
using namespace std;

void f1(int n)
{
    n++;
    cout << "f1: n = " << n << endl;
}

void f2(int &n) // 引用参数
{
    n++;
    cout << "f2: n = " << n << endl;
}

class A
{
public:
    int m_count;
    std::list<string> m_lst;
public:
    A(){ cout << "A::A 构造" << endl;}
    A(const A &a){ 
        cout << "A::A(const &A) 拷贝构造" << endl;
        m_count = a.m_count;
        m_lst = a.m_lst;
    }
    virtual ~A (){cout << "A::~A 析构" << endl;}
};

void f3(A *a) // 对象指针
{
    a->m_count = 10;
    a->m_lst.push_back("today ");
    a->m_lst.push_back("is a good day");
}

void f4(A &a) // 对象引用
{
    a.m_count = 10;
    a.m_lst.push_back("today ");
    a.m_lst.push_back("is a good day");
}

void f5(A a) // 对象传值
{
    a.m_count = 10;
    a.m_lst.push_back("today ");
    a.m_lst.push_back("is a good day");
}

int main()
{
    int n = 0;

    // 复制方式传入
    thread t1(f1, n);
    t1.join();
    cout << "t1 main: n = " << n << endl;

    // 引入方式传入
    thread t2(f2, ref(n));
    t2.join();
    cout << "t2 main: n = " << n << endl;

    // 指针方式传入对象
    cout << "===== 指针传值 ===== " << endl;
    A *a1 = new A();
    thread t3(f3, a1);
    t3.join();
    cout << "t3 main: count = " << a1->m_count << " list:" << a1->m_lst.size() << endl;

    // 引用传值
    cout << "===== 引用传值 ===== " << endl;
    A a2;
    thread t4(f4, ref(a2));
    t4.join();
    cout << "t4 main: count = " << a2.m_count << " list:" << a2.m_lst.size() << endl;

    // 对象传值
    cout << "===== 对象传值 ===== " << endl;
    A a3;
    thread t5(f5, a3);
    t5.join();
    cout << "t5 main: count = " << a3.m_count << " list:" << a3.m_lst.size() << endl;
}

  • 带锁的多线程计算传参
#include <iostream>
#include <thread>
#include <mutex>
 
void step(std::mutex &mutex, int loop, int &counter) {
    for (int i = 0; i < loop; i++) {
        mutex.lock();
        ++counter;
        mutex.unlock();
    }
}
int main() {
    std::thread threads[5];
    std::mutex mutex;
    int counter = 0;
 
    for (std::thread &thr: threads) {
        // thread的方法传递引用或bind的时候，必须用ref来进行引用传递，否则就是浅拷贝。
        // ref是函数模板,用来构建一个reference_wrapper对象并返回，该对象拥有传入的elem变量的引用
        thr = std::thread(step, std::ref(mutex), 1000, std::ref(counter));
    }
    for (std::thread &thr: threads) {
        thr.join();
    }
 
    // 输出：5000，如果step中调用的是try_lock，则此处可能会<5000
    std::cout << counter << std::endl;
 
    return 0;
}
//: g++ -std=c++11 main.cpp -pthread

# 互斥锁mutex

  • C++11自带互斥锁,在<mutex>中定义;
  • 使用lock加锁,unlock解锁,必须成对出现,不能连续两次lock或unlock;(windows临界区可以)
  • throw/return等均可能造成未unlock;
  • mutex声明后可马上使用,不需要自己初始化
#include <iostream>
#include <thread>
#include <mutex>
#include <vector>
#include <list>
using namespace std;

// 放消息
class A
{
public:
    void inMsgQueue()
    {
        for (int i = 0; i < 30; i++)
        {
            mymutex.lock();
            msgQueue.push_back(i);
            mymutex.unlock();
            cout << "in,total:" << i << endl;
            std::this_thread::sleep_for(std::chrono::milliseconds(300));
        }
    }
    void outMsgQueue()
    {
        for (int i = 0; i < 30; i++)
        {
            mymutex.lock();
            if (!msgQueue.empty())
            {
                int recv = msgQueue.front();
                msgQueue.pop_front();
                cout << "out:" << recv << endl;
            }
            else
            {
                cout << "outMsgQueue waiting, empty queue" << endl;
            }
            mymutex.unlock();
            std::this_thread::sleep_for(std::chrono::milliseconds(500));
        }
    }
private:
    std::list<int> msgQueue;
    std::mutex mymutex;
};

int main()
{
    A myobj;
    std::thread th1(&A::inMsgQueue, &myobj);
    std::thread th2(&A::outMsgQueue, &myobj);
    
    th1.join();
    th2.join();
}

# std::lock_guard

  • 生命周期开始,自动加锁,结束自动解锁
  • 同一个生命周期范围内,不允许重复锁,会导致锁死;(局部生命周期或函数调用都不行)
  • 使用了lock_guard,不要使用裸的lock和unlock
// 例子1
void inMsgQueue()
{
    for (int i = 0; i < 30; i++)
    {
        {
            std::lock_guard<std::mutex> mylockguard(mymutex);
            msgQueue.push_back(i);
            cout << "in,total:" << i << endl;
        }
        std::this_thread::sleep_for(std::chrono::milliseconds(300));
    }
}

// 例子2
void test(){
    std::lock_guard<std::mutex> mylockguard2(mymutex);
}
void inMsgQueue()
{
    for (int i = 0; i < 30; i++)
    {
        {
            std::lock_guard<std::mutex> mylockguard(mymutex);
            // test(); // 同一个生命周期有多个同一个mutex会锁死
            msgQueue.push_back(i);
            cout << "in,total:" << i << endl;
        }
        test(); // ok
        std::this_thread::sleep_for(std::chrono::milliseconds(300));
    }
}

# 死锁

  • 两把以上的锁才可能形成死锁
  • 锁的顺序不同,且互相依赖就导致了死锁（lock有关系,unlock关系无关）
  • std::lock_guard也存在死锁情况;
#include <iostream>
#include <thread>
#include <mutex>
#include <vector>
#include <list>
using namespace std;

// 放消息
class A
{
public:
    void inMsgQueue()
    {
        for (int i = 0; i < 30; i++)
        {
            mymutex1.lock();
            mymutex2.lock();
            msgQueue.push_back(i);
            mymutex1.unlock();
            mymutex2.unlock();
            cout << "in,total:" << i << endl;
        }
    }
    void outMsgQueue()
    {
        for (int i = 0; i < 30; i++)
        {
            mymutex2.lock();
            mymutex1.lock(); // lock的顺序必须想法才能促成死锁
            if (!msgQueue.empty())
            {
                int recv = msgQueue.front();
                msgQueue.pop_front();
                cout << "out:" << recv << endl;
            }
            else
            {
                cout << "outMsgQueue waiting, empty queue" << endl;
            }
            mymutex1.unlock();
            mymutex2.unlock();
        }
    }

private:
    std::list<int> msgQueue;
    std::mutex mymutex1;
    std::mutex mymutex2;
};

int main()
{
    A myobj;
    // 传入第二个参数是对象的引用指针(this指针),保证两个线程操作同一对象;后续的参数对应function的其他参数;
    std::thread th1(&A::inMsgQueue, &myobj);
    std::thread th2(&A::outMsgQueue, &myobj);

    th1.join();
    th2.join();
}

# 死锁-解决

  • 保证锁的顺序
  • 使用std::lock()模板(谨慎使用,实际代码中少用,仔细考虑锁的范围)
    • 使用std:lock()模板同时锁住多个互斥量(2个以上)
    • lock中mutex的顺序可以不同,模板会保证同时锁住或不锁住mutex;
    • 没有std::unlock()模板,锁住后,仍需逐一unlock锁;
    • 使用时lock()中mutex顺序可以不同;
void inMsgQueue()
{
    for (int i = 0; i < 30; i++)
    {
        std::lock(mymutex1, mymutex2);
        msgQueue.push_back(i);
        mymutex1.unlock(); // unlock顺序也无关,但不能遗忘
        mymutex2.unlock();
        cout << "in,total:" << i << endl;
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
    }
}
void outMsgQueue()
{
    for (int i = 0; i < 50; i++)
    {
        std::lock(mymutex2, mymutex1); // 顺序可以不同
        if (!msgQueue.empty())
        {
            int recv = msgQueue.front();
            msgQueue.pop_front();
            cout << "out:" << recv << endl;
        }
        else
        {
            cout << i << " outMsgQueue waiting, empty queue" << endl;
        }
        mymutex1.unlock();
        mymutex2.unlock();
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
    }
}

# unique_lock

  • 在部分场景可用来取代lock_guard,更灵活但相对耗性能;
  • unique_lock第一个参数为mutex,第二个参数可以有多种: | 参数 | 说明 | |------------------|----------------------------------------------------| | std::adopt_lock | 使用方式同lock_guard,锁住情况下,其他线程会锁住等待 | | std::try_to_lock | 锁住情况下,尝试解锁,但不会堵塞等待锁 | | std::defer_lock | 延迟锁定的锁 |

# try_to_lock例子

  • 使用时不能先对mutex进行lock
  • 尝试锁后根据own_lcok()函数核实是否取到锁,并进行后续逻辑处理,特点是不阻塞
void inMsgQueue()
{
    for (int i = 0; i < 10; i++)
    {
        {
            std::lock_guard<mutex> guard1(mymutex1, std::adopt_lock);
            msgQueue.push_back(i);
            cout << "in,total:" << i+1 << ",size:" << msgQueue.size() << endl;
            std::this_thread::sleep_for(std::chrono::seconds(5));
            cout << "unlock ..." << endl;
        }
    }
}
void outMsgQueue()
{
    for (int i = 0; i < 50; i++)
    {
        {
            std::this_thread::sleep_for(std::chrono::seconds(1));
            std::unique_lock<mutex> unique_lock1(mymutex1, std::try_to_lock);
            if (unique_lock1.owns_lock()){
                if (msgQueue.size() == 0) {
                    cout << i << " empty queue." << endl;
                    continue;
                }
                int recv = msgQueue.front();
                msgQueue.pop_front();
                cout << i << " out,recv:" <<  recv << ",size:" << msgQueue.size() << endl;
            }
            else
            {
                cout << i << " obtain unique_lock1 fail" << endl;
            }
        }
        
    }
}

# defer_lock

  • unique_lock 成员函数（针对defer_lock） | 函数 | 说明 | |----------|------------------------------------------------| | lock | 锁定(支持不手动unlock也能自动解锁) | | unlock | 手动解锁,避免锁包含的区域过大,相对灵活 | | try_lock | 尝试加锁返回加锁的结果,不堵塞 | | release | 返回unique_lock管理的mutex对象指针并释放所有权 |

  • lock/unlock临时放开锁

void getMsg(int i){
    if (msgQueue.size() == 0) {
        cout << "[out]empty queue," << i << endl;
        return;
    }
    int recv = msgQueue.front();
    msgQueue.pop_front();
    cout << "[out]recv:" << recv << " " << i << ",size:" << msgQueue.size() << endl;
}

            }

            unique_lock1.lock();    
            getMsg(i); 

        }
    }

  • try_lock();

注意: try_lock()不能与普通lock混用,可能出现try_lock经常性拿到锁的场景,而lock始终拿不到锁;

void inMsgQueue()
{
    for (int i = 0; i < 100; i++)
    {
        std::unique_lock<mutex> unique_lock1(mymutex1, std::defer_lock);
        if (unique_lock1.try_lock())                {
            msgQueue.push_back(i);
            cout << "[in]total:" << i + 1 << ",size:" << msgQueue.size() << endl;
        }
        else
        {
            cout << "[in]not obtain lock" << i + 1 << ",size:" << msgQueue.size() << endl;
        }
        std::this_thread::sleep_for(std::chrono::seconds(1));
        cout << "[in]unlock ..." << endl;
    }
}
void getMsg(int i)
{
    if (msgQueue.size() == 0)        {
        cout << "[out]empty queue," << i << endl;
        return;
    }
    int recv = msgQueue.front();
    msgQueue.pop_front();
    cout << "[out]recv:" << recv << " " << i << ",size:" << msgQueue.size() << endl;
}
void outMsgQueue()
{
    for (int i = 0; i < 100; i++)
    {
        std::this_thread::sleep_for(std::chrono::seconds(1));
        std::unique_lock<mutex> unique_lock1(mymutex1, std::defer_lock);
        if (unique_lock1.try_lock())     {
            getMsg(i);
        }    else       {
            cout << "[out]not obtain lock" << i + 1 << ",size:" << msgQueue.size() << endl;
        }
    }
}
