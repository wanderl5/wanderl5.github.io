# valgrind使用

valgrind可检测出 Valgrind时建立动态分析工具的框架。它有一系列用于调试分析的工具：

  • Memcheck：用于检测内存泄漏或者错误访问。它帮助c和c++的程序更正确;(常用,这是默认工具)
  • Helgrind: 用于分析多线程;(常用,赋值竞争)
  • Cachegrind：用于分析cache和分支预测。它帮助程序执行得更快;
  • Callgrind: 用于函数调用的分析;
  • DRD: 也用于分析多线程。与Helgrind类似，但是用不同的分析技术，所以可以检测不同的问题。
  • Massif：用于分析堆。它帮助程序精简内存的使用。
  • SGcheck：检测栈和全局数组溢出的实验性工具，它和Memcheck互补使用。

注意: 编译程序时用-g，这样编译后的文件包含调试信息，那Memcheck的分析信息中就包含正确的行号。最好使用-O0的优化等级，使用-O2及以上的优化等级使用时可能会有问题。

# 源码

  • 编译: g++ main.cpp -g -O0 -lpthread

#include <pthread.h>
#include <iostream>
#include <unistd.h>
int var = 0;
void *child_fn(void *arg)
{
	int *tid = (int*)arg;
	/* Unprotected relative to parent */ /* this is line 6 */
	for (int i = 0; i < 10; i++)
	{
		var++;
		unsigned int microseconds = 3 * 1000 * 1000; // 微秒,3秒
		usleep(microseconds);
		int *tmp = new int[30]; // 奇数次会有内存泄露
		if (i % 2 == 0){
			delete []tmp;
		}
		std::cout << "[" << *tid << "]sleep microseconds:" << microseconds
			<< ",var:" << var << std::endl;
	}
	delete tid;
	return NULL;
}

int main(void)
{
	pthread_t child1, child2;
	int* tid1 = new int();
	*tid1 = 1;
	int* tid2 = new int();
	*tid2 = 2;
	pthread_create(&child1, NULL, child_fn, (void *)tid1);
	pthread_create(&child2, NULL, child_fn, (void *)tid2);
	
	pthread_join(child1, NULL);
	pthread_join(child2, NULL);
	std::cout << "all done" << std::endl;

	return 0;
}

{
    int*x = (int *)malloc(10 * sizeof(int));
    x[10]= 0; // 非法访问
    //problem 1: heap block overrun
}   //problem 2: memory leak -- x not freed

int main(void)
{
    f();
    return 0;
