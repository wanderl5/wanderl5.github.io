MQ8安装(Linux)

# MQ 8.0 Linux 下安装

摘自： https://www.pianshen.com/article/6757190044/

# 下载和上传安装文件

  • 上传 WS_MQ_V8..0.0.2_LINUX_ON_X86_64_I.tar.gz 到临时安装目录下,如~/tmp
  • 解压，解压后文件目录名称为server
tar -zxvf WS_MQ_V8..0.0.2_LINUX_ON_X86_64_I.tar.gz

或者
gzip WS_MQ_V8..0.0.2_LINUX_ON_X86_64_I.tar.gz
tar xvf WS_MQ_V8..0.0.2_LINUX_ON_X86_64_I.tar

apt-get install rpm

  • 创建MQ用户和用户组
groupadd mqm

vi /etc/group
将mqm:!:1000:mqm 后面加上root (mqm:!:1000:mqm,root)

  • 创建WebSphere MQ 文件系统
mkdir /opt/mqm

  • 添加MQ接受许可
cd ~/tmp
./mqlicense.sh -accept

  • 安装 WebSphere MQ 服务器
rpm -ivh MQSeriesRuntime-*.rpm
rpm -ivh MQSeriesSDK-8*.rpm
rpm -ivh MQSeriesServer-*.rpm
1
2
3
  • 安装 WebSphere MQ 样本程序（其中包括amqsput、amqsget、amqsgbr和amqsbcg等）
#样本程序安装在/opt/mqm/samp/bin中
rpm -ivh MQSeriesSamples-*.rpm
1
2
  • 检查系统相关参数是否符合要求，修改相应参数
su mqm -c/opt/mqm/bin/mqconfig
1

  • 检查结果可能如下:(mqconfig提示:进程能打开文件数不够(soft limits))

  • 修改/etc/sysctl.conf中设置：

#fs.file-max = 524288

#---配置内核参数---
echo "fs.file-max = 524288" >>/etc/sysctl.conf
#echo "kernel.threads-max = 32768" >>/etc/sysctl.conf

# TODO: 此处会有提示: sysctl: setting key "fs.file-max", ignoring: Read-only file system
sysctl -p /etc/sysctl.conf

#---更改 limits 值---
echo "mqm hard nofile 10240" >>/etc/security/limits.conf
echo "mqm soft nofile 10240" >>/etc/security/limits.conf
echo "mqm hard nproc 4096" >>/etc/security/limits.conf
echo "mqm soft nproc 4096" >>/etc/security/limits.conf

  • 检查远程连接
# 放入消息
[qe@E193114 ~]$cd /opt/mqm/samp/bin/
# 格式: amqsput ql QM
[qe@E193114 /opt/mqm/samp/bin]$amqsput MX.MXIO.DEFAULT.RQ QEQMGR_I1
Sample AMQSPUT0 start
target queue is MX.MXIO.DEFAULT.RQ
111
^C 


# get message
[qe@E193114 /opt/mqm/samp/bin]$amqsget MX.MXIO.DEFAULT.RQ QEQMGR_I1
Sample AMQSGET0 start
message <1234>
no more messages
Sample AMQSGET0 end


直接连接会报:
2035 MQRC_NOT_AUTHORIZED

runmqsc 后执行以下命令：解决x86服务器连接性通道连接不上问题
ALTER QMGR CHLAUTH (DISABLED)

以下两条在7.5win版本下有问题:(可能9.0可以,待核实)
ALTER QMGR CONNAUTH('')
REFRESH SECURITY TYPE(CONNAUTH)
